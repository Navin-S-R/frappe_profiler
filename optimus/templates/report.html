How to read this report
  • Start with Findings — that's the punch list of what to fix. The Observations sub-list under it is FYI signal (framework-level issues, system pressure) with no concrete fix you can ship.
  • The per-action table below lists every request the session captured, sorted by duration; the background-jobs section lists the jobs those requests enqueued (captured and analyzed too).
  • The database-tables section shows where DB time went, with an index candidate per table; the tables written to line at the top of it lists every table the session wrote — writes are cheap, so they rank low by time. The slowest-queries section covers your own app's code only.
  • The server-resource and frontend panels show CPU / memory / queue pressure and browser timings — only when captured.
  • The full-recordings and queries-per-action sections (collapsed) hold the raw SQL, headers, form data and stack traces — for deep dives.
  • Hover any file.py:123 callsite to open it in VS Code. Anything starting with tab is a database table; the DocType is the part after tab.
